# EA-AOL Overview

EA-AOL is intended to be a standalone project repository for runtime, language, and tooling work that belongs to the EA-AOL boundary and does not need to share ownership with the rest of the Antigravity research tree.

## Repository Intent

- Keep the project legible as an independent system.
- Separate design documents, contracts, implementation, and verification from the start.
- Avoid turning the repository into a generic holding area for unrelated experiments.

## What This Repository Should Eventually Contain

- Runtime or compiler entrypoints that define the project surface.
- Project-specific contracts and invariants.
- Small examples or demos that explain the intended execution model.
- Tests that prove the project boundary has not silently drifted.

## What This Repository Should Not Become

- A scratch area for unfinished experiments that have not earned a project identity.
- A duplicate of `project-manuals` for process documentation.
- A mirror of `cognitive-lab` for general research implementation.
